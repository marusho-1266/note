# Obsidianタグの検証・最適化プロンプト

以下のプロンプトを使用して、Cursorに既存のObsidianタグの検証と最適化を指示できます。

## タグ一貫性検証プロンプト

```
@00_Inbox_Clip フォルダ内のファイルに付与されているタグを分析し、以下の観点で一貫性を検証してください:

1. tag_rule.mdに定義されたルール違反がないか
2. 類似タグや重複タグがないか
3. 階層タグが適切に使用されているか
4. 不適切なタグがないか
5. 「#inbox」タグが残っていないか確認し、存在する場合は削除対象として記録

各ファイルのタグを抽出し、問題点があれば指摘してください。また、全体的なタグ使用状況の統計（使用頻度順）を表示してください。

## 検証結果の出力形式
1. タグ使用統計（頻度順）
2. ルール違反タグのリスト
3. 最適化提案（類似タグの統合など）
4. ファイル別の問題タグ
5. 「#inbox」タグが残っているファイル一覧
6. @01_Stockに移動すべきファイル一覧
```

## タグクリーンアッププロンプト

```
@00_Inbox_Clip フォルダ内のファイルのタグをクリーンアップするために、以下の作業を行ってください:

1. 以下の問題タグを修正:
   [ここに問題タグのリストを入力]

2. 以下のタグを統合:
   [ここに統合すべきタグペアを入力]
   例: #ai と #artificial-intelligence → #ai に統一

3. 以下のタグを階層化:
   [ここに階層化すべきタグを入力]
   例: #python → #tech/programming/python

4. すべてのファイルから「#inbox」タグを削除

各ファイルを処理し、変更内容を報告してください。変更はFrontmatterのtagsプロパティに対して行います。

処理が完了したら、タグ付けが完了したファイルを@01_Stockフォルダに移動してください。
```

## タグカテゴリ分析プロンプト

```
@00_Inbox_Clip フォルダ内のファイルに付与されたタグを分析し、主要カテゴリごとにグループ化してください。

1. 技術・IT関連タグ
2. ビジネス・EC関連タグ
3. 生産性・ツール関連タグ
4. 情報タイプタグ
5. その他のカテゴリ

分析結果から、以下の洞察を提供してください:
- 最も頻出するトピックカテゴリ
- 不足しているタグカテゴリ
- タグ運用の効率化提案
- 「#inbox」タグが残っているファイル一覧

結果は視覚的に理解しやすい形式で出力してください。

分析が完了したら、適切にタグ付けされたファイルを@01_Stockフォルダに移動するための手順も提案してください。
```
