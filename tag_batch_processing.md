# Obsidianタグのバッチ処理プロンプト

以下のプロンプトを使用して、Cursorに複数のObsidianノートへのタグ処理を一括で指示できます。

## バッチタグ適用プロンプト

```
00_Inbox_Clip フォルダ内のすべてのファイルに対して、以下のバッチ処理を行ってください:

1. 各ファイルの内容を分析し、tag_rule.mdに基づいて適切なタグを自動生成
2. 既存のtags:プロパティがあるファイルは、それを更新
3. tags:プロパティがないファイルには、新しくFrontmatterを追加
4. 各ファイルのタグを最大5つに制限
5. 「#inbox」タグがある場合は削除
6. 処理が完了したファイルを01_Stockフォルダに移動

処理中は以下の点に注意してください:
- 各ファイルの内容を十分に理解してからタグを付与する
- ルールに従ってタグの命名規則を厳守する
- 階層タグが適切な場合は積極的に使用する

## 出力形式
各ファイルの処理結果を以下の形式で簡潔に報告してください:

ファイル名: [変更したファイル名]
適用タグ: [適用したタグリスト]
移動先: 01_Stock/[ファイル名]
```

## 特定トピックのバッチタグ付けプロンプト

```
00_Inbox_Clip フォルダ内の[特定トピック]に関連するすべてのファイルを特定し、適切なタグを付与してください。

1. 対象トピック: [DNS/EC/AI/Obsidianなど]
2. 適用すべき基本タグ: [#dns/#ec/#ai/#obsidianなど]
3. 関連する階層タグ候補: [#tech/network/dns など]

各ファイルの内容を分析し、指定されたトピックに関連するファイルを識別してください。関連ファイルには、基本タグに加えて、内容に応じた詳細なタグを付与してください。

既存のタグから「#inbox」を削除し、処理が完了したファイルを01_Stockフォルダに移動してください。

## 出力形式
- 関連ファイルリスト
- 各ファイルに適用したタグ
- 処理したファイル数の統計
- 各ファイルの移動先パス
```

## 経時的なタグ更新プロンプト

```
@00_Inbox_Clip フォルダ内のファイルについて、最近追加された新しいファイルを優先してタグを付与・更新してください。

以下の条件で処理するファイルを選択します:
1. Frontmatterにcreatedプロパティがあり、[日付]以降のファイル
2. tags:プロパティがないファイル
3. tags:プロパティに"#inbox"タグのみが含まれるファイル

各ファイルの内容を分析し、tag_rule.mdに基づいて適切なタグを自動生成してください。既存のtagsプロパティを更新し、ない場合は新たに追加します。すべてのファイルから「#inbox」タグを削除してください。

タグ付けが完了したファイルは01_Stockフォルダに移動してください。

## 出力形式
処理したファイル数と、各ファイルの適用タグを簡潔に報告してください。また、各ファイルの移動先も記載してください。
```
